Задание 2.

Доработать программу для получения данных с мосбиржи.

1. Устранить замечания по результатам Core review.
2. Вынести функциональность работы с Мосбиржей в отдельный модуль moex.py (сетевые обращения, парсинг и тп).
3. Отрефакторить основной модуль приложения, чтобы для получения данных с мосбиржи он использовал функции из moex.py.
4. Добавить дополнительный режим работы приложения - отслеживание изменений.

Режим "Отслеживание изменений":
При запуске приложения с опцией "--watch" включается режим отслеживания изменений.
В этом режиме приложение запрашивает информацию с сервера с заданной периодичностью ("--refresh" секунд, по умолчанию 60).
При наличии обновлений они записываются в файл и в stdout выводится сообщение, что данные обновились.
При отсутствии обновлений - сообщение, что обновлений нет.
Формат сообщения в stdout:
"[2020-03-12 15:15:15] <текст сообщения>"

В режиме "--watch" программа работает бесконечно до тех пор пока её не выключат с помощью "Ctrl+C" (SIGINT).
Обратите внимание, программа должна штатно завершиться в этом случае (не упасть с Exception), закрыть файлы и т.д.

Решение необходимо загрузить в git (scm.x5.ru, доступен по VPN).

Вам необходимо будет создать репозиторий через интерфейс GitLab и организовать его следующим образом:

Ветка master с одним коммитом - файл README.
Ветка dev содержит ваше решение.
Из ветки dev в ветку master необходимо создать Merge Request (через интерфейс GtLab).


В созданных Merge Request мы проведем персональные Code review в таком же формате, как делаем это в рамках промышленной разработки.
